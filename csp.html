College Board AP Daily 3.1 - 3.14 Notes

Variables and Assignments: A variable is an abstraction used to store values in a program. Some languages provide types like numbers, Booleans, lists, and strings, which are referenced using variables. Variable values can be changed, and the "‚Üê" operator is used to assign a value.

Data Abstraction: An index is a method to reference elements in a list or string using natural numbers. Data abstraction separates the abstract properties of a data type from its specific details. Lists organize items like strings or numbers to form data abstraction, making programs easier to develop and maintain.

Mathematical Expressions: An algorithm is a set of steps designed to solve a specific task. Algorithms can be represented in natural language, diagrams, or pseudocode. They are built using sequences, selection, and iteration.

Strings: Strings are sequences of characters used for representing text in programming. A substring is a part of an existing string. Strings are used for names, words, or sentences, and you can manipulate them by combining, splitting, or searching.

Boolean Expressions: A Boolean is either true or false. It can have a value of "1" (true) or "0" (false). Boolean operators like AND, OR, and NOT are often used in conditionals and loops.

Conditionals: Conditionals allow code to make decisions based on conditions. They use `if`, `else if`, and `else` to execute different code based on Boolean expressions.

Nested Conditionals: Nested conditionals occur when a conditional contains another conditional inside it. They are used for handling more complex decision-making processes.

Iteration: Iteration allows a portion of an algorithm to repeat. Loops execute a block of code multiple times based on a condition, such as in `REPEAT UNTIL(condition)` or `FOR` loops. If the condition evaluates as true initially, the loop does not execute.

Developing Algorithms: Algorithms can be developed from an idea, by combining existing algorithms, or by modifying current solutions. These sequences of steps solve specific problems and often incorporate patterns for efficiency.

Lists: Lists store multiple elements that can be accessed using their index positions. Algorithms working with lists often use iteration to traverse and process their elements, making them a critical part of data abstraction in programming.

Binary Search: Binary search algorithms eliminate half the data with each round of splitting. Data must be sorted to use a binary search method effectively.

Calling and Developing Procedures: Procedures allow solving large problems by breaking them into smaller sub-problems. Procedural abstraction enables programmers to modify or fix procedures without affecting the entire program. Procedures often require parameters and are called with arguments. Once a procedure terminates, the program resumes from where it was called.

Libraries: Libraries provide pre-written code that can be imported to create new programs. APIs explain how to use the procedures in a library.

Random Values: Random number generation in a program can yield different results every time it is run.

Simulations: Simulations simplify complex objects or events to investigate real-world scenarios. They may omit details and carry a risk of oversimplification or bias based on creator choices.

Algorithmic Efficiency: Efficiency determines how often statements execute. Algorithms with polynomial efficiency or lower are considered reasonable, while those with exponential efficiency are not. Approximate solutions using heuristics may be required for certain problems.

Undecidable Problems: An undecidable problem can be solved for some cases, but no algorithm exists to solve it in all cases.
